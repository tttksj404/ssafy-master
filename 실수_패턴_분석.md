## 1. Visited 체크 실수 (가장 빈번)

### 1-A. 빈 셀/중립 셀 처리 누락
```python
# 3184bfs정석여러변수.py — 첫 시도에서 '.' 셀을 무시함
if grid[nr][nc] == 'o':
    visited[nr][nc] = True
    queue.append(...)
# ← '.' 셀에 대한 처리가 없어서 BFS가 중간에 끊김
```
**교훈**: BFS 조건문 작성 시 벽이 아닌 모든 셀 타입(빈칸, 시작점, 목적지 등)을 빠짐없이 처리해야 함.

### 1-B. visited를 리스트로 사용 → O(N) 탐색
```python
# 1219 DFS — visited=[] + if current not in visited
visited = []                    # ← O(N) lookup
visited.append(current)
if current not in visited:      # 매번 전체 순회
```
**수정**: `visited = [False] * N` 또는 `visited = set()` 사용. `01_자료구조/21464`에서 본인이 직접 list→set 효율화를 경험했는데, DFS 쪽에서는 아직 적용 안 된 파일이 있음.

### 1-C. 글로벌 큐 재사용 (위험한 패턴)
```python
# 2468, 2667, 1926, 3184 등 다수
queue = deque()  # ← 모듈 레벨 전역 변수

def bfs(x, y):
    queue.append((x, y))  # 이전 BFS에서 안 빠진 데이터가 남아있을 수 있음
```
**교훈**: BFS 함수 안에서 `queue = deque()` 로 매번 새로 만들기. 전역 큐는 잔여 데이터로 오답 발생 가능.

## 2. R/C (행/열) 혼동 (치명적)

### 2-A. 입력 파싱에서 W와 H 뒤바뀜
```python
# 5656벽돌깨기.py
W, H = map(int, input().split())
brick_map = [list(map(int, input().split())) for _ in range(W)]
#                                                         ^^^ H여야 함
```
**교훈**: 입력받을 때 `for _ in range(행의 수)` 인지 반드시 확인. W(폭)=열, H(높이)=행.

### 2-B. 배열 선언과 접근 순서 불일치
```python
visited = [[0]*C for _ in range(R)]  # R행 C열
visited[c][r]  # ← 거꾸로 접근 → IndexError (R≠C일 때)
```
**규칙**: 선언: `[_ for _ in range(R)]` → 접근: `[i][j]` 에서 i=행(R), j=열(C). 항상 R→C 순서 통일.

## 3. 함수 호출 vs 인덱싱 혼동

```python
# 5650핀볼.py — 첫 시도에서 5곳에서 동일 실수
def ordinary_wall(nr, nc):
    ...

# 호출할 때:
ordinary_wall[nr][nc]  # ← TypeError: function은 subscript 안 됨
ordinary_wall(nr, nc)  # ← 이게 맞음
```
**패턴**: 함수를 `()` 대신 `[]`로 호출하는 실수가 핀볼 문제에서 5번 반복됨. 함수명 뒤에는 항상 `()`.

## 4. Off-by-One 에러

### 4-A. BFS 거리 시작값
```python
# 1697 숨바꼭질 — visited로 거리 추적
visited[start] += 1  # ← 시작점 거리가 1이 됨 (0이어야 함)
# 나중에 print(bfs(N,K) - 1) 로 땜빵 → 근본 해결이 아님
```
**교훈**: BFS 거리는 시작점 = 0부터. `visited[start] = 0` 또는 별도 `dist[]` 배열 사용.

### 4-B. 경계 조건에 데드코드
```python
# 2105 디저트카페
for i in range(N):
    for j in range(N):
        if j==0 or i==N or j==N or i==N-1:  # i==N, j==N은 절대 True가 안 됨
            continue                          # range(N)이면 최대 N-1
```
**수정**: `i==N` → `i==N-1`, `j==N` → `j==N-1`.

## 6. 탐색 기준 노드 혼동

```python
# 여러 BFS/DFS 파일에서 반복
cur = queue.popleft()

for nxt in graph[start]:  # ← start가 아니라 cur이어야 함!
    ...
```
**규칙**: `pop`/`popleft` 한 변수가 지금의 기준. 매개변수나 초기값을 그대로 쓰면 제자리걸음.

## 9. N-Queen 구조적 설계 실수

```python
# NQUEEN.py 첫 시도 — 3가지 동시 실수
depth -= 1          # ← 루프 안에서 매개변수 직접 수정 → 반복 오염
visited[nr][nc]     # ← 범위 밖 접근 가능 (nr/nc 증가 후 체크)
flag = True/False   # ← 전역 플래그로 충돌/완료 동시 관리 → 의미 불명확
```
**수정 (두 번째 시도)**: 행 단위 배치 + `col[]`/`diag1[]`/`diag2[]` 배열로 O(N!) → 정석.

## ★ 문제 풀기 전 최종 체크리스트

```
□ 입력: R(행)과 C(열) 구분했나? range(R) 맞나?
□ 배열: 크기 넉넉한가? (0~99면 100개, 0~100000이면 100001개)
□ visited: 리스트 말고 bool 배열이나 set 쓰고 있나?
□ visited: 큐에 넣기 직전에 체크하고 있나? (꺼낼 때 X)
□ visited: 원복 필요한 문제인가? (모든경로/순열 = O, 도달/최단 = X)
□ BFS 큐: 함수 안에서 새로 만들고 있나? (전역 X)
□ pop 후: 방금 꺼낸 변수를 기준으로 다음 탐색하고 있나?
□ 범위: for range(N)이면 최대 N-1. 경계조건에 N 쓰면 데드코드
□ 전파: 중간 셀 전부 순회하고 있나? (끝점만 체크 X)
□ 함수: () 로 호출하고 있나? [] 아님
```
