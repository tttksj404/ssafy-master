# Step 4: 이진 탐색 & 그리디 마스터 가이드

이 단계의 목표는 10억, 100억 개의 방대한 데이터 범위에서 '결정 문제'로 바꿔 답을 찾아내는 파라메트릭 서치와, 지역적 최적해로 전체 최적해를 찾는 그리디/투포인터 기법을 익히는 것입니다.

### 1. 파라메트릭 서치(Parametric Search)의 4단계 공식

"**최댓값**의 **최솟값**" 혹은 "**최솟값**의 **최댓값**"을 구하라는 문제는 대부분 파라메트릭 서치입니다.

1.  **결정 문제로 변환**: "답이 X일 때, 문제의 조건을 만족하는가?" 라는 질문으로 바꾼다.
2.  **탐색 범위 설정**: `start`, `end`를 정한다. (최소 가능 값 ~ 최대 가능 값)
3.  **`check(mid)` 함수 구현**: `mid`를 답으로 가정했을 때, 조건을 만족하는지(True/False) 판별하는 함수를 짠다.
4.  **이진 탐색**: `check` 함수의 결과에 따라 `start`와 `end`를 조절하며 답의 범위를 좁힌다.

```python
# 파라메트릭 서치 기본 템플릿
start = 1
end = 1000000000
ans = 0

while start <= end:
    mid = (start + end) // 2

    if check(mid): # 조건을 만족한다면?
        ans = mid # 일단 답이 될 수 있으니 저장
        start = mid + 1 # 더 좋은 답(최댓값)을 찾아 오른쪽으로
        # end = mid - 1   # 더 좋은 답(최솟값)을 찾아 왼쪽으로
    else: # 조건을 만족하지 못하면?
        end = mid - 1   # 답의 크기를 줄여야 함
        # start = mid + 1 # 답의 크기를 키워야 함
```
- **예시**: 1654번(랜선 자르기), 2110번(공유기 설치), 1300번(K번째 수)

### 2. 투 포인터(Two Pointers) & 슬라이딩 윈도우

- **상황**: **정렬된 배열**에서 두 수의 합/차를 보거나, 특정 길이의 **연속된 구간**을 다룰 때.
- **투 포인터**: 양쪽 끝에서 시작해 서로를 향해 좁혀오는 방식. (예: 2470번 용액)
- **슬라이딩 윈도우**: 고정된 크기의 창문을 한 칸씩 밀고 가는 방식. (예: 2559번 수열)

두 기법 모두 O(N^2)의 복잡도를 O(N)으로 줄여주는 아주 강력한 도구입니다.
