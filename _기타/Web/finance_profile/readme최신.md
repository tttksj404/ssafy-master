# 반응형 Profile 페이지 구현

## 어려웠던 점과 해결
- html 구현에서 여러 스켈레톤을 활용했고, 어려움이 있을때 마다 찾아봤다 
- 비슷한 스켈레톤을 웹서핑하면 찾을 수 있고, 이를 이용하였다. 

## 프로젝트 개요
- 프로젝트명: 금융 상품 비교 애플리케이션 프로필 페이지 구현
- 구현 일자: 2026-03-06
- 사용 기술: HTML, CSS (반응형 Media Query)
   

## 구현 항목
- F301: 기본 정보 수정 페이지에서 프로필 사진, 자기소개, 투자 성향 출력 및 수정 UI 구현
- F302: 회원 기본 정보(회원번호, ID, Email, Nickname, 나이, 자산 총액) 입력 UI 구현
- F303: 상단 네비게이션 바로 기본 정보 페이지/포트폴리오 페이지 이동 구현
- F304: 포트폴리오 수정 페이지(회원번호, ID, 대출 성향, 목표 대출액) 구현
- F305: 화면 너비 768px 기준 반응형 레이아웃 구현
- NF301: 구현 과정 및 학습 내용 문서화

## 파일 구성
- `index.html`: 기본 정보 수정 페이지
- `portfolio.html`: 포트폴리오 수정 페이지
- `style.css`: 공통 스타일 및 반응형 규칙

## 실행 방법
1. `index.html` 또는 `portfolio.html` 파일을 브라우저로 연다.
2. 네비게이션 바를 통해 두 페이지를 이동한다.
3. 브라우저 폭을 768px 미만으로 줄여 반응형 레이아웃을 확인한다.

## 구현 과정에서 학습한 점
- 큰 화면에서는 3열 그리드로, 작은 화면에서는 1열로 변경하는 방식으로 구조를 단순화했다.
- `readonly`, `email`, `number` 타입을 사용해 입력 제약을 UI 레벨에서 반영했다.
- 공통 컴포넌트(네비게이션, 폼, 버튼)를 하나의 CSS로 관리해 중복 스타일을 줄였다.
